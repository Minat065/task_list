# 回答文作成システム 要件定義書

**作成日**: 2026-03-17
**ステータス**: ドラフト

---

## 1. プロジェクト概要

### 1.1 背景
商品に問題があった場合、原因調査を行い、その結果をお客様へ回答する業務が存在する。
現状はExcelベースのマニュアル集から適切な回答パターンを手動で選択し、回答文を作成している。
このプロセスをシステム化し、正確性の向上と作業効率の改善を図る。

### 1.2 現状の課題
- マニュアル集から適切な回答パターンを探す手間がかかる
- 条件に応じた追記の漏れリスクがある
- 回答の品質が担当者のスキルに依存する
- 回答履歴の管理・検索が困難

### 1.3 ゴール
条件を選択・入力することで、適切な回答文を自動生成するシステムを構築する。

---

## 2. 業務フロー

```
[問題発生] → [原因調査] → [システムで条件入力] → [回答文自動生成] → [確認・編集] → [お客様へ送付]
```

### 2.1 ユーザー操作フロー
1. 容器の種類を選択
2. 問題点を選択（大分類 → 中分類 → 小分類）
3. 程度・条件を入力/選択
4. システムが該当するマニュアル（テンプレート）を自動で特定
5. 回答文（調査結果/製造工程/まとめ）を自動生成
6. 条件に応じた追記が自動付与される
7. ユーザーが確認・必要に応じて編集
8. 最終成果物を出力

---

## 3. 機能要件

### 3.1 マスタ管理機能
- **容器マスタ**: 容器の種類を管理（6〜20種類、追加可能）
- **問題分類マスタ**: 階層構造で問題点を管理（深さ制限なし）
  - 例: 漏れ・破損 → 亀裂 → 小さな傷 / 大きな傷 / 裂傷 / 押し込み
  - 末端の分類まで選択すると、対応するテンプレートが特定される
- **主回答文テンプレート**: 末端問題分類ごとのベース回答文を管理
- **サブ回答文テンプレート**: 条件に応じて主回答文の前後に付与するテキストを管理

### 3.2 回答文生成機能
- 容器×末端問題分類から該当する主回答文を自動特定
- 3パート構成の回答文を生成:
  - **調査結果**: 調査内容と判明した原因
  - **製造工程**: 関連する製造工程の説明
  - **まとめ**: 結論と今後の対応
- 条件に該当するサブ回答文を主回答文の前または後ろに付与
  - 例:「流通経路による場合」→ サブ回答文を追記
  - 例:「当たったものが明確な場合」→ サブ回答文を追記
- 生成後の手動編集機能

### 3.3 出力機能
- 画面上でのプレビュー
- ファイル出力（形式は後続フェーズで決定）

### 3.4 履歴管理機能
- 過去の回答文を検索・参照
- どの条件で生成されたかの記録

## 4. DB設計（コア）

### 4.1 ER図（概念）

```
[containers]               [problem_categories]
容器マスタ                   問題分類マスタ（階層構造・深さ制限なし）
 |                            |
 +----------------------------+
               |
        [manual_entries]
        マニュアルエントリ
        （容器×末端問題分類 → テンプレート群への紐付け）
               |
        [response_templates]
        主回答文テンプレート
        （調査結果/製造工程/まとめ の3パート）

        [sub_responses]
        サブ回答文テンプレート
        （条件に応じて主回答文の前or後に付与）

        [response_history]
        回答履歴
```

### 4.2 テーブル定義

#### 4.2.1 containers（容器マスタ）
| カラム名 | 型 | 説明 |
|---------|------|------|
| id | INT PK | 容器ID |
| name | VARCHAR | 容器名（例: 缶、ボトル、パウチ） |
| description | TEXT | 備考 |
| sort_order | INT | 表示順 |
| is_active | BOOLEAN | 有効フラグ |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### 4.2.2 problem_categories（問題分類マスタ）
階層構造を自己参照で実現する。

| カラム名 | 型 | 説明 |
|---------|------|------|
| id | INT PK | 分類ID |
| parent_id | INT FK(self) | 親分類ID（NULLならルート） |
| depth | INT | 階層の深さ（1=大分類, 2=中分類, 3=小分類） |
| name | VARCHAR | 分類名 |
| description | TEXT | 備考 |
| sort_order | INT | 同階層内の表示順 |
| is_active | BOOLEAN | 有効フラグ |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### 4.2.3 manual_entries（マニュアルエントリ）
容器×末端問題分類の組み合わせを管理するハブテーブル。

| カラム名 | 型 | 説明 |
|---------|------|------|
| id | INT PK | エントリID |
| container_id | INT FK | 容器ID |
| problem_category_id | INT FK | 問題分類ID（末端の分類を指定） |
| name | VARCHAR | エントリ名（管理用） |
| description | TEXT | 備考 |
| is_active | BOOLEAN | 有効フラグ |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### 4.2.4 response_templates（回答テンプレート）
各パートのテンプレート本文を管理。

| カラム名 | 型 | 説明 |
|---------|------|------|
| id | INT PK | テンプレートID |
| manual_entry_id | INT FK | マニュアルエントリID |
| section_type | ENUM | パート種別: 'investigation' / 'process' / 'summary' |
| body | TEXT | テンプレート本文（プレースホルダ対応） |
| sort_order | INT | 同セクション内の表示順 |
| is_active | BOOLEAN | 有効フラグ |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### 4.2.5 sub_responses（サブ回答文テンプレート）
条件に応じて主回答文の前または後ろに付与するテキストを管理。

| カラム名 | 型 | 説明 |
|---------|------|------|
| id | INT PK | サブ回答文ID |
| template_id | INT FK | 対象の主回答文テンプレートID |
| condition_label | VARCHAR | 条件の表示ラベル（例: "流通経路による場合"） |
| position | ENUM | 付与位置: 'before' / 'after' |
| body | TEXT | サブ回答文の本文 |
| sort_order | INT | 同位置内の表示順 |
| is_active | BOOLEAN | 有効フラグ |
| created_at | TIMESTAMP | 作成日時 |
| updated_at | TIMESTAMP | 更新日時 |

#### 4.2.7 response_history（回答履歴）
生成された回答の履歴を記録。

| カラム名 | 型 | 説明 |
|---------|------|------|
| id | INT PK | 履歴ID |
| manual_entry_id | INT FK | 使用したエントリID |
| container_id | INT FK | 容器ID |
| problem_category_id | INT FK | 問題分類ID |
| generated_text | TEXT | 生成された回答文全文 |
| edited_text | TEXT NULL | 編集後の回答文（編集があった場合） |
| created_by | VARCHAR | 作成者 |
| created_at | TIMESTAMP | 作成日時 |

#### 4.2.7 response_history_subs（履歴×適用したサブ回答文）
履歴にどのサブ回答文が適用されたかを記録。

| カラム名 | 型 | 説明 |
|---------|------|------|
| id | INT PK | ID |
| response_history_id | INT FK | 履歴ID |
| sub_response_id | INT FK | 適用したサブ回答文ID |

## 5. 具体例（飲料会社を想定）

以下、**「350ml缶に亀裂があり、小さな傷だった」** というクレーム対応を例にデータの流れを示す。

| id | name | description |

| 41 | 4 | 2 | 穴あき |
| 42 | 4 | 2 | 変形 |
| 43 | 4 | 2 | 接合不良 |
| 400 | 40 | 3 | **小さな傷** |
| 401 | 40 | 3 | **大きな傷** |

### 5.3 manual_entries（マニュアルエントリ）

**容器×末端問題分類** の組み合わせがエントリになる：

| id | container_id | problem_category_id | name |
|----|-------------|--------------------|----|
| 1 | 1 (350ml缶) | 400 (小さな傷) | **缶・亀裂・小さな傷** |
| 2 | 1 (350ml缶) | 401 (大きな傷) | 缶・亀裂・大きな傷 |
| 3 | 1 (350ml缶) | 402 (裂傷) | 缶・亀裂・裂傷 |
| 4 | 1 (350ml缶) | 403 (押し込み) | 缶・亀裂・押し込み |
| 5 | 2 (500mlPET) | 400 (小さな傷) | PET・亀裂・小さな傷 |

### 5.4 response_templates（回答テンプレート）

エントリ1「缶・亀裂・小さな傷」に紐づく3パートのテンプレート：

| id | manual_entry_id | section_type | body |
|----|----------------|-------------|------|
| 1 | 1 | **investigation** | お届けいたしました{{product_name}}に傷が確認された件につきまして、調査結果をご報告いたします。現品を確認したところ、缶体表面に微小な傷が認められました。傷の形状および方向から、製造ラインの搬送過程において缶同士の接触により生じたものと判断いたしました。なお、内容物への影響はございません。 |
| 2 | 1 | **process** | 当該製品の製造工程は以下の通りです。缶体成形 → 洗浄 → 充填 → 密封 → 殺菌 → 冷却 → 外観検査 → 梱包。外観検査工程ではカメラによる自動検査を実施しておりますが、今回の傷は検出閾値以下の微小なものでありました。 |
| 3 | 1 | **summary** | このたびはご不快な思いをおかけしましたことを深くお詫び申し上げます。搬送ラインのガイド調整を実施し、缶同士の接触を低減する対策を講じました。 |

### 5.5 sub_responses（サブ回答文テンプレート）

主回答文に対して、条件に応じて前or後ろに付与するサブ回答文：

| id | template_id | condition_label | position | body | sort_order |
|----|------------|----------------|----------|------|-----------|
| 1 | 1 | 流通経路による場合 | after | 流通過程における温度管理および取り扱い状況を確認いたしましたところ、輸送時の振動により缶体に負荷がかかった可能性が認められました。 | 1 |
| 2 | 1 | 当たったものが明確な場合 | after | なお、接触対象物の成分を分析した結果、{{contact_object}}由来の痕跡が確認されました。 | 2 |
| 3 | 3 | 同一ロットで複数報告がある場合 | after | 同一製造ロットの他製品についても確認を行い、必要に応じて回収等の措置を検討いたします。 | 1 |
| 4 | 2 | 流通経路による場合 | before | なお、本件は製造工程内ではなく、流通過程で生じた可能性が高いと判断しております。 | 1 |

**動作例（サブ回答文の付与イメージ）:**

- サブ1: 「流通経路による」→ はい → 調査結果の主回答文の**後ろ**に流通経路の説明を追記
- サブ2: 「当たったものが明確」→ はい → 調査結果の主回答文の**後ろ**に接触対象の分析結果を追記
- サブ3: 「同一ロットで複数報告」→ はい → まとめの主回答文の**後ろ**に回収検討の文を追記
- サブ4: 「流通経路による」→ はい → 製造工程の主回答文の**前**に流通起因である旨を追記

### 5.6 生成される回答文の例

**入力:** 350ml缶 × 漏れ・破損 → 亀裂 → 小さな傷

**サブ回答文の条件:** 「流通経路による場合」→ **はい** / 「当たったものが明確な場合」→ いいえ

---

> **【調査結果】**
> お届けいたしました緑茶「まるごと茶葉」に傷が確認された件につきまして、調査結果をご報告いたします。現品を確認したところ、缶体表面に微小な傷が認められました。傷の形状および方向から、製造ラインの搬送過程において缶同士の接触により生じたものと判断いたしました。流通過程における温度管理および取り扱い状況を確認いたしましたところ、輸送時の振動により缶体に負荷がかかった可能性が認められました。
>
> **【製造工程】**
> なお、本件は製造工程内ではなく、流通過程で生じた可能性が高いと判断しております。当該製品の製造工程は以下の通りです。缶体成形 → 洗浄 → 充填 → 密封 → 殺菌 → 冷却 → 外観検査 → 梱包。外観検査工程ではカメラによる自動検査を実施しておりますが、今回の傷は検出閾値以下の微小なものでありました。
>
> **【まとめ】**
> このたびはご不快な思いをおかけしましたことを深くお詫び申し上げます。搬送ラインのガイド調整を実施し、缶同士の接触を低減する対策を講じました。

*(「流通経路による」→ はい → サブ1が調査結果の後ろに、サブ4が製造工程の前に付与。「当たったものが明確」→ いいえ → サブ2は付与されない)*

---

### 5.7 データの流れまとめ

```
ユーザーが選択:
  容器:     [350ml缶]         → containers.id = 1
  大分類:   [漏れ・破損]       → problem_categories.id = 4
  中分類:   [亀裂]             → problem_categories.id = 40
  小分類:   [小さな傷]         → problem_categories.id = 400

        ↓ マッチング

  manual_entries: id=1（缶×小さな傷）が該当

        ↓ テンプレート取得

  response_templates: エントリ1に紐づく3パートの主回答文を取得
  sub_responses: 各主回答文に紐づくサブ回答文を取得

        ↓ ユーザーにサブ回答文の条件を質問

  「流通経路による場合」→ はい
  「当たったものが明確な場合」→ いいえ

        ↓ 主回答文 + 該当サブ回答文(前or後)を結合 + プレースホルダ置換

  最終回答文を生成
```

## 6. DB設計のポイント

### 6.1 階層構造の表現
- `problem_categories` テーブルの自己参照（`parent_id`）で任意の深さの階層を表現
- `depth` カラムでUI表示時の階層判定を高速化
- 問題の種類ごとに階層の深さが異なっても対応可能（例: 亀裂は4階層、変形は3階層で終わる等）

### 6.2 シンプルなテンプレート特定
- `manual_entries` は容器×末端問題分類の直接マッチング
- 条件マスタ不要 — 問題分類の階層自体が条件の役割を果たす
- 末端分類まで選べば、対応するテンプレートが一意に決まる

### 6.3 サブ回答文による条件付き追記
- `sub_responses` で主回答文に追加するテキストブロックを管理
- `position`（before/after）で主回答文の前か後ろかを指定するだけのシンプルな仕組み
- ユーザーにはYes/No形式で条件を質問し、該当するサブ回答文を付与

### 6.4 テンプレートの構成
- `response_templates` で3パート（調査結果/製造工程/まとめ）を個別管理
- 本文にプレースホルダ（例: `{{product_name}}`, `{{equipment_name}}`）を埋め込み、動的に置換

## 9. 用語集

| 用語 | 意味 |
|------|------|
| 容器 | 商品の容器種別（缶、ボトル等） |
| 問題分類 | 問題点の階層的な分類体系 |
| マニュアルエントリ | 容器×末端問題分類の組み合わせに対応するマニュアル項目 |
| 主回答文 | 末端問題分類で決まるベースの回答文（プレースホルダを含む） |
| サブ回答文 | 条件に応じて主回答文の前後に付与するテキストブロック |
| パート | 回答文の構成要素（調査結果/製造工程/まとめ） |
