# Slide Type Classifier Skill

## 役割
プレゼンの各スライド内容を分析し、最適なスライド型を判定する。

## 入力
ユーザーからのプレゼン指示（テーマ、スライド構成案）

## 出力JSON形式

```json
{
  "slides": [
    {
      "slide_number": 1,
      "type": "TYPE_01",
      "title": "スライドタイトル",
      "purpose": "製品のビジュアル紹介",
      "needs_3d": true,
      "notes": "メイン製品画像を左に大きく配置"
    },
    {
      "slide_number": 2,
      "type": "TYPE_02",
      "title": "3つの強み",
      "purpose": "特徴の比較",
      "needs_3d": false,
      "notes": "3カラムで各強みを対比"
    }
  ]
}
```

## 型判定ルール

### TYPE_01: ビジュアル＋解説型
**選択条件:**
- 1つの主要ビジュアル（製品写真、3Dモデル、概念図）が中心
- 補足情報が3つ程度のブロックに分かれる
- 製品紹介、機能説明、ビフォーアフター

### TYPE_02: 3列比較型
**選択条件:**
- 3つの並列な要素を比較・対比
- プラン比較、特徴3選、ステップ概要
- 各要素に見出し＋アイコン＋箇条書き

### TYPE_03: グリッド一覧型
**選択条件:**
- 4件以上のアイテムを一覧表示
- カテゴリ別整理、ポートフォリオ、チーム紹介
- 各アイテムが均等な情報量

### TYPE_04: フェーズ/プロセス型
**選択条件:**
- 時系列・段階的な流れ
- ロードマップ、導入プロセス、開発フェーズ
- 各フェーズに詳細説明が必要

## needs_3d 判定
- 製品の外観紹介 → true
- 建築・インテリア → true
- 抽象概念・データ → false
- 人物・チーム → false
